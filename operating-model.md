# Operating Model

## How We Work
You define:
- business direction
- content style preference
- examples you like
- approval/rejection on outputs

I handle:
- persona design
- prompt engineering
- workflow design
- consistency rules
- generation briefs
- content planning
- asset naming and organization
- next-step instructions

## Review Loop
For every generation batch:
1. keep only outputs that preserve identity
2. reject drift immediately
3. tag what changed: pose, outfit, background, hair, makeup
4. stabilize before expanding

## Asset Rules
- never overwrite approved anchor images
- keep versioned names
- separate anchors from experiments

## Folder Intent
- `assets/` approved visuals
- `character/` structured identity definitions
- `scripts/` setup scripts
- root docs for project direction and workflows

## Immediate Workflow
1. install model
2. run first portrait workflow
3. curate best output
4. create reference pack
5. generate 5 close variants
6. generate 5 scene variants
7. move to profile and first posts
